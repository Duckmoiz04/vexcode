### 3.3. Lựa chọn công nghệ

Dưới đây là luận giải cho các lựa chọn công nghệ trong hệ thống, dựa trên yêu cầu vận hành local, chi phí thấp và khả năng mở rộng.

*Bảng 3.5. Lựa chọn công nghệ và lý do*

| Công nghệ | Lựa chọn | Lý do chính |
|-----------|----------|-------------|
| **Static Analysis Engine** | OpenGrep (Semgrep fork) | Pattern-matching đa ngôn ngữ, chạy hoàn toàn local, mã nguồn mở, không phụ thuộc cloud |
| **AST & Knowledge Graph** | Tree-sitter + GitNexus | Tree-sitter parse nhanh, incremental. GitNexus xây dựng đồ thị tri thức từ AST, tracing call graph và execution flow |
| **LLM Integration** | OpenRouter + NVIDIA NIM | Lớp trừu tượng hóa provider: không phụ thuộc một LLM duy nhất, hỗ trợ cả thương mại (Claude, GPT) và mã nguồn mở (Llama, Qwen, DeepSeek) |
| **Backend Framework** | Node.js 20 (ESM) + Express 4.x | Non-blocking I/O phù hợp streaming SSE, runtime phổ biến, hệ sinh thái npm lớn |
| **Frontend Framework** | React 19 + TypeScript + Vite | TypeScript đảm bảo type-safety, Vite build nhanh, React hỗ trợ component-based UI phức tạp |
| **UI Components** | CodeMirror 6 + Recharts | CodeMirror nhẹ hơn Monaco, phù hợp hiển thị code snippet. Recharts cho biểu đồ thống kê |
| **Lưu trữ** | File JSON | Dữ liệu phi cấu trúc, mỗi lần quét là snapshot độc lập, không cần index phức tạp |
| **AI Pipeline** | Subprocess (Node gọi Python) | Tách biệt runtime, engine Python chạy độc lập, dễ kiểm thử và mở rộng |
| **Taxonomy** | ISO/IEC 25010 | Tiêu chuẩn quốc tế về chất lượng phần mềm, phù hợp định hướng KTPM |
| **Testing** | Vitest (JS/TS) + pytest (Python) | Vitest tương thích Jest API, nhanh nhờ Vite. pytest với fixture linh hoạt |

**Tại sao không chọn các công nghệ khác:**

*Bảng 3.6. Công nghệ thay thế và lý do không chọn*

| Công nghệ thay thế | Lý do không chọn |
|--------------------|-----------------|
| SonarQube | Quá nặng, yêu cầu server riêng, không phù hợp triển khai local |
| ESLint / Pylint | Chỉ hoạt động trong một ngôn ngữ duy nhất, không phát hiện lỗi bảo mật đa ngôn ngữ |
| PostgreSQL / SQLite | Overhead so với file JSON cho use case snapshot độc lập |
| Next.js (React server) | Dư thừa cho dashboard client-side, tăng độ phức tạp deploy |
| Tự fine-tune LLM | Chi phí cao, không khả thi với tài nguyên hiện có |
