### 3.4. Thiết kế chi tiết

#### 3.3.1. Thiết kế lưu trữ dữ liệu

Hệ thống sử dụng lưu trữ file thay vì cơ sở dữ liệu quan hệ, vì kết quả quét là dữ liệu phi cấu trúc dạng snapshot độc lập. Định dạng JSON cho phép linh hoạt thay đổi schema và dễ dàng chia sẻ, sao lưu.

**Cấu trúc thư mục lưu trữ:**

```
~/.vexcode/
├── scans/
│   ├── <scan_id>/
│   │   ├── report.json              # Kết quả quét hoàn chỉnh
│   │   ├── raw_findings.json        # Kết quả từ Scanner (SARIF)
│   │   ├── enriched_findings.json   # Kết quả sau Enricher
│   │   └── resolved_findings.json   # Kết quả sau Resolver
│   └── scan_index.json              # Chỉ mục các lần quét
├── sessions/
│   └── <scan_id>_<finding_id>.json  # Lưu bộ nhớ hội thoại Ask AI
├── bin/
│   └── opengrep*                    # OpenGrep binary đã tải
├── backups/                         # Sao lưu mã nguồn trước khi áp dụng fix
└── config.json                      # Cấu hình người dùng
```

Mỗi finding được gán một ID duy nhất dựa trên hash(file_path, line_number, rule_id) để phục vụ theo dõi vòng đời qua nhiều lần quét. File scan_index.json lưu lịch sử trạng thái của từng finding theo thời gian.

#### 3.3.2. Thiết kế giao diện (UI/UX)

Dashboard của VexCode được thiết kế theo phong cách modern SaaS với bố cục ba khu vực: thanh điều hướng (Navbar), khu vực nội dung chính và khu vực trạng thái.

**Các trang chính:**

- **Dashboard tổng quan**: Biểu đồ phân bố severity, biểu đồ category, thẻ thống kê, top file lỗi nặng nhất, điểm sức khỏe dự án, so sánh cross-scan.
- **Findings**: Bảng danh sách lỗi với filter (severity, category, status), tìm kiếm toàn văn bản, sắp xếp và phân trang.
- **Chi tiết lỗi**: CodeMirror hiển thị code với highlight vị trí lỗi, panel phân tích AI (classification, explanation, suggestion, diff), chat panel Ask AI, action bar (Apply Fix, Rollback, đánh dấu trạng thái).
- **Cấu hình**: Chọn model AI, nhập API key (OpenAI, Anthropic, Google, 9router, NVIDIA), tùy chỉnh tham số.

**Components chính:** Layout, Sidebar, Header, CodeInspector, ChatPanel, FilterPanel, SettingsDrawer, ScanModal, và các biểu đồ dashboard.

#### 3.3.3. Thiết kế API

Backend Node.js cung cấp các REST API endpoint sau:

*Bảng 3.4. Danh sách REST API endpoints*

| Phương thức | Endpoint | Chức năng |
|------------|----------|-----------|
| POST | `/api/scan` | Bắt đầu quét |
| POST | `/api/scan/cancel` | Hủy quét |
| GET | `/api/scan/stream` | SSE stream tiến trình |
| GET | `/api/reports` | Danh sách dự án |
| GET | `/api/reports/:project` | Danh sách báo cáo |
| GET | `/api/report/:project/:id` | Chi tiết báo cáo (phân trang) |
| GET | `/api/report/:project/:id/sarif` | SARIF sidecar |
| POST | `/api/re-resolve` | Chạy lại AI |
| POST | `/api/finding/.../status` | Cập nhật trạng thái |
| GET | `/api/file-content` | Đọc nội dung file |
| POST | `/api/open-in-ide` | Mở trong IDE |
| GET/POST | `/api/config` | Đọc/ghi cấu hình |
| GET/PUT | `/api/settings/ai` | Cấu hình AI |
| POST | `/api/chat` | Gửi tin nhắn AI |
| POST | `/api/apply` | Áp dụng sửa lỗi |
| POST | `/api/rollback` | Hoàn tác sửa lỗi |
| GET | `/api/models` | Danh sách model |

Kết quả quét được stream qua SSE với bốn event type: progress (tiến độ), finding (kết quả từng lỗi), complete (hoàn tất), error (lỗi).

#### 3.3.4. Cơ chế bảo mật

Hệ thống vận hành trên local, áp dụng các lớp bảo mật sau:

- **Chống directory traversal**: isPathSafe() chuẩn hóa và kiểm tra đường dẫn đầu vào
- **Authentication**: API server lắng nghe trên localhost, hỗ trợ Bearer token
- **Rate limiting**: 100 req/15 phút global, 10 req/15 phút cho scan
- **Subprocess safety**: list arguments, không dùng shell string, timeout 120s
- **Error sanitization**: Không leak stack trace hay đường dẫn nội bộ
- **File permission**: .env với 0o600, keys mặc định rỗng
- **Input validation**: Request body giới hạn 100KB
