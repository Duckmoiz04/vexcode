### 3.6. Triển khai hệ thống

Phần này trình bày chi tiết việc hiện thực hóa ba thành phần chính của hệ thống — Python Engine, Node.js Backend và React Frontend — dựa trên kiến trúc đã thiết kế ở các mục trước.

**3.6.1. Python Engine**

<!-- TODO: Chèn Hình 4.2. Kiến trúc module Python Engine -->

Python Engine là thành phần cốt lõi chịu trách nhiệm thực thi pipeline phân tích mã nguồn. Engine được tổ chức theo mô hình module, mỗi module đảm nhận một giai đoạn trong pipeline.

**Module Scanner — engine/core/scanner.py**

Scanner quét mã nguồn bằng OpenGrep. Module này triển khai:

- Hàm `ensure_opengrep()` kiểm tra và tự động tải OpenGrep binary từ GitHub Releases nếu chưa tồn tại, hỗ trợ Windows (x86), macOS (x86/arm64) và Linux (x86/arm64).
- Hàm `run_opengrep(target, language)` xây dựng câu lệnh OpenGrep với tham số phù hợp, thực thi qua `subprocess.run()`, thu thập đầu ra SARIF dạng JSON.
- Hàm `parse_sarif_output(raw)` phân tích cấu trúc SARIF, trích xuất danh sách findings với các trường: file, line, column, rule_id, message, severity.
- Ánh xạ phần mở rộng file sang ngôn ngữ tương ứng.

Cơ chế fallback: Khi OpenGrep không khả dụng hoặc quét thất bại, Scanner trả về mock findings để đảm bảo pipeline vẫn chạy được.

**Module Pipeline — engine/pipeline/**

Pipeline điều phối chung, thực thi tuần tự bốn giai đoạn:

1. `Scanner.run()`: Gọi OpenGrep, parse SARIF, phân loại findings theo ISO 25010.
2. `Enricher.run()`: Với mỗi finding, đọc file code, trích xuất hàm chứa lỗi và các hàm liên quan.
3. `Resolver.run()`: Gom nhóm findings theo file, gọi API LLM, parse kết quả JSON.
4. `Reporter.run()`: Tổng hợp kết quả, xây dựng báo cáo JSON với summary và findings.

Pipeline hỗ trợ hai chế độ: `--mock-scan` và `--mock-ai`.

**Module AI Resolver — engine/resolver/**

Resolver xử lý giao tiếp với LLM:
1. Xây dựng prompt với cấu trúc: code + ngữ cảnh + ISO 25010 rule.
2. Gọi API qua httpx đến OpenRouter / NVIDIA NIM, retry 3 lần, timeout 60s.
3. Parse JSON, kiểm tra trường bắt buộc, ghi log nếu lỗi.

**Module ISO 25010 Taxonomy — engine/config/iso25010_taxonomy.py**

Module này định nghĩa ánh xạ từ rule_id của OpenGrep sang các đặc tính ISO 25010:

```python
TAXONOMY_MAP = {
    "python.lang.security.audit.dangerous-exec": ("security", "CWE-95"),
    "python.lang.security.audit.hardcoded-password": ("security", "CWE-798"),
    # ... ~100 mapping rules
}
```

Mỗi finding được gán ID duy nhất (hash của file-line-rule) để theo dõi vòng đời.

**Cơ chế Install OpenGrep — engine/opengrep_installer.py**

Script tự động tải OpenGrep binary phù hợp với nền tảng từ GitHub Releases, lưu vào `~/.vexcode/bin/`.

**3.6.2. Node.js Backend**

<!-- TODO: Chèn Hình 4.3. Kiến trúc Node.js Backend -->

Node.js Backend đóng vai trò trung gian điều phối, được xây dựng bằng Express.js.

**Cấu trúc module:**

```
packages/cli/src/
├── server/index.js      # Express server, CORS, routes
├── routes/              # scan, findings, chat, models, config, apply
├── services/            # scan-manager, findings-store, chat-store
└── bridge/
    └── engine-bridge.js # Gọi Python Engine qua subprocess
```

**Module Bridge** là quan trọng nhất: nhận yêu cầu scan, spawn Python Engine, đọc stdout từng dòng (JSON lines), emit sự kiện, xử lý timeout.

**Module SSE**: Mỗi scan có một SSE channel riêng. Khi Bridge emit event, Server ghi vào response stream. Client dùng `EventSource` để nhận.

**Module lưu trữ**: findings-store.js và chat-store.js dùng file system, lưu tại `~/.vexcode/scans/` và `~/.vexcode/sessions/`.

**Express Server**: Port 3000 (mặc định), CORS cho Vite dev server, static files từ `public/`, error handling middleware.

**3.6.3. React Frontend**

<!-- TODO: Chèn Hình 4.4. Component tree React Frontend -->

Frontend Dashboard được xây dựng với React 19 + TypeScript + Vite.

**Cấu trúc component:**

- **components/**: Layout, Navbar, ScanForm, ProgressBar, StatCard, SeverityChart, CategoryChart, FindingsTable, FindingDetail, MonacoViewer, ChatPanel, StatusBadge, ModelSelector
- **hooks/**: useScan (scan state + SSE), useChat (hội thoại), useReports (báo cáo)
- **pages/**: DashboardPage (thống kê, biểu đồ), FindingsPage (filter, search), FindingDetailPage (code + AI + chat), SettingsPage (API keys, model)
- **context/**: AppContext quản lý state toàn cục (currentScan, findings, chatHistory, models, settings)
- **services/**: api.ts — API client (fetch + SSE)
