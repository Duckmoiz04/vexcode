# Chương 3: Phân tích và thiết kế hệ thống

## 3.3. Thiết kế kiến trúc tổng thể

### 3.3.1. Nguyên tắc thiết kế

Hệ thống được xây dựng trên ba nguyên tắc chính:

**Hybrid SAST-first, AI辅助.** Hệ thống dùng SAST (Static Application Security Testing) làm lớp phát hiện lỗi chính, AI chỉ đóng vai trò đánh giá và đề xuất sửa. Cách tiếp cận này tận dụng độ chính xác của SAST cho các lỗi đã biết (pattern-based), đồng thời dùng AI để giải quyết false positive và cung cấp mã sửa. Việc phát hiện bằng SAST trước còn giảm đáng kể số lượng token cần gửi đến AI — yếu tố quyết định chi phí và thời gian.

**Graceful degradation ở mọi tầng.** Mỗi thành phần phụ thuộc (GitNexus, AI provider, scanner) đều có cơ chế fallback độc lập. Thiếu GitNexus thì bỏ qua enrichment; thiếu AI thì dùng mock resolution; scanner lỗi thì dùng mock scan. Hệ thống luôn trả về kết quả, không fail hoàn toàn.

**Tách biệt ba nền tảng.** Node.js cho CLI và API server; Python cho pipeline phân tích; React cho web dashboard. Mỗi tầng độc lập, giao tiếp qua JSON qua stdin/stdout hoặc REST API.

Tầng CLI/API là điểm vào duy nhất. Node.js kiểm tra an toàn đường dẫn, spawn Python subprocess, lắng nghe progress events và chuyển tiếp đến web dashboard qua SSE. Tầng phân tích thực hiện toàn bộ pipeline. Web dashboard chỉ đọc kết quả qua REST API và hiển thị.

Lựa chọn ba tầng riêng biệt vì: Python có hệ sinh thái phong phú cho phân tích tĩnh (AST parsing, complexity metrics, subprocess management); Node.js xử lý I/O bất đồng bộ tốt cho API server; React phù hợp cho dashboard tương tác với real-time progress.

### 3.3.3. Luồng dữ liệu chính

Một phiên quét (scan session) đi qua năm giai đoạn tuần tự:

```
1.  Người dùng nhập đường dẫn dự án (CLI hoặc Dashboard)
2.  Node.js kiểm tra workspace boundary, spawn Python process
3.  Python pipeline thực hiện năm giai đoạn tuần tự:
    Scan → Enrich → AI Resolution → Report & Quality Gate
4.  Mỗi giai đoạn ghi progress JSON ra stdout, Node.js parse
    và chuyển thành SSE events đến web dashboard
5.  Python ghi file JSON kết quả vào ~/.vexcode/reports/
    (kèm SARIF sidecar nếu không dùng --no-sarif)
6.  Node.js đọc file JSON, trả về cho dashboard
7.  Người dùng duyệt findings, xem diff, apply/reject/ignore
```

### 3.3.4. Bảng tổng hợp kiến trúc ba tầng

| Tầng | Công nghệ | Vai trò | Output |
|------|-----------|---------|--------|
| Giao diện | React 19 + TypeScript + Vite SPA | Hiển thị findings, diff view, tương tác người dùng | DOM |
| CLI / API | Node.js ESM + Express | Điểm vào, REST API, SSE progress, quản lý process | REST API, SSE |
| Phân tích | Python 3.12 + pipeline | Scan → Enrich → AI Resolution → Report & Quality Gate | JSON, SARIF |

Kiến trúc ba tầng giúp mỗi tầng phát triển và kiểm thử độc lập. Tầng Python không phụ thuộc vào Node.js; web dashboard có thể kết nối bất kỳ backend nào cung cấp đúng REST API.
