# Chương 3: Phân tích và thiết kế hệ thống

## 3.3. Thiết kế kiến trúc tổng thể

Hệ thống được tổ chức theo kiến trúc ba tầng (three-tier architecture), mỗi tầng sử dụng một nền tảng công nghệ khác nhau.

### 3.3.1. Sơ đồ kiến trúc

Tầng CLI/API (Node.js) là điểm vào duy nhất của hệ thống, cung cấp cả command-line interface và REST API server. Tầng phân tích (Python) thực hiện pipeline xử lý tám giai đoạn. Tầng giao diện (React) chạy trên trình duyệt và giao tiếp với server qua REST API.

Kiến trúc này được chọn vì ba lý do. Thứ nhất, tách biệt rõ ràng giữa presentation, business logic và data processing giúp dễ bảo trì và mở rộng. Thứ hai, Python có hệ sinh thái phân tích dữ liệu phong phú và các thư viện machine learning mạnh, phù hợp cho tầng xử lý. Thứ ba, Node.js xử lý I/O bất đồng bộ tốt, phù hợp cho tầng API và CLI cần phản hồi nhanh.

### 3.3.2. Luồng xử lý chính

Một phiên quét (scan session) đi qua các bước sau:

1. Người dùng nhập đường dẫn dự án (qua CLI hoặc web).
2. Node.js kiểm tra đường dẫn an toàn, spawn Python subprocess.
3. Python pipeline thực hiện tám phase tuần tự: **Scan → Enrich → Complexity → Dedup → Classify → Naming Audit → AI Resolve → Report**. Mỗi phase nhận đầu ra của phase trước và bổ sung dữ liệu.
4. Mỗi phase ghi log progress ra stdout dưới dạng JSON struct, Node.js đọc và chuyển tiếp thành SSE events đến web dashboard để hiển thị thanh tiến độ real-time.
5. Khi hoàn tất, Python ghi file JSON kết quả vào thư mục reports (kèm SARIF sidecar nếu không tắt).
6. Node.js đọc file JSON và trả về cho web dashboard.
7. Người dùng duyệt findings, xem đề xuất sửa lỗi và áp dụng nếu muốn.
