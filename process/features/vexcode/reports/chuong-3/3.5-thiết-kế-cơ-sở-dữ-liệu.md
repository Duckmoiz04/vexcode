## 3.5. Thiết kế cơ sở dữ liệu

Hệ thống không sử dụng cơ sở dữ liệu quan hệ. Toàn bộ dữ liệu được lưu dưới dạng file JSON trong thư mục ***~/.vexcode/reports/***. Mỗi project có một thư mục riêng, bên trong là các file JSON đánh dấu thời gian cho mỗi lần scan. Cách tiếp cận này giúp đơn giản hóa việc triển khai (không cần cài đặt DB server), dễ dàng backup và cho phép so sánh giữa các lần quét bằng cách so sánh trực tiếp các file JSON với nhau.

### 3.5.1. Finding: Đơn vị dữ liệu trung tâm

**Finding** là thực thể trung tâm của toàn bộ hệ thống, được biểu diễn dưới dạng Python ***dict*** với các trường chính:

| Trường | Ý nghĩa |
|--------|---------|
| ***id*** | Định danh SHA-1(***file|line|rule_id***), 12 ký tự hex |
| ***file***, ***line*** | Vị trí phát hiện |
| ***rule_id***, ***severity*** | Quy tắc và mức độ |
| ***ast_context*** | Ngữ cảnh AST từ GitNexus |
| ***status*** | Trạng thái xử lý (***open***/***applied***/***false_positive***/***ignored***) |
| ***scan_status*** | Trạng thái liên scan (***new***/***persisting***/***resolved***/***regressed***) |

**Cơ chế định danh (***id***):** Định danh được tính bằng SHA-1 của tổ hợp ***file:line:rule_id***, lấy 12 ký tự hex đầu tiên. Cơ chế này đảm bảo:
- Cùng một lỗi luôn có cùng id qua các lần quét.
- Khi developer sửa lỗi và dòng code thay đổi, id cũng thay đổi.
- Khi re-scan, id cho phép phát hiện lỗi mới (NEW), lỗi đã sửa (RESOLVED) và lỗi tái phát (REGRESSED).

### 3.5.2. Report: Cấu trúc báo cáo

Mỗi lần scan tạo ra một file JSON do ***assemble_report*** (trong ***reporter.py***) assemble, dùng làm đầu vào cho dashboard và đầu ra lưu trữ. Cấu trúc cố định gồm:
- ***scanner***, ***timestamp***, ***target_path*** — metadata phiên scan
- ***findings*** — mảng kết quả sau enrich, dedup, resolve
- ***ai_resolutions*** — map ***rule_id*** → resolution dict
- ***git_state***, ***metrics*** — trạng thái Git và độ phức tạp mã nguồn

Ghi file an toàn bằng **atomic write**: ghi .***tmp*** rồi ***os.replace***, tránh file cắt xén khi crash. Ngoài JSON, hệ thống xuất thêm:
- **SARIF 2.1.0** (.***sarif***) cho CI/CD
- **Markdown** (.***md***) báo cáo người đọc, do ***export_markdown*** tạo

### 3.5.3. AI Resolution: Đề xuất sửa lỗi

Mỗi resolution là output của pipeline 3-stage (Analyze → Fix → Review) trong ***core/ai_resolver.py***, lưu trong ***report.ai_resolutions*** theo ***rule_id*** để các findings cùng rule dùng chung.

**Luồng xử lý:**
1. **Analyze** — AI phân loại: ***confirmed***, ***hotspot***, ***false_positive***. Nếu ***false_positive*** hoặc ***hotspot*** (không cần sửa chi tiết) thì dừng sớm, tiết kiệm token.
2. **Fix** — Sinh ***remediation_code*** từ code thực tế xung quanh lỗi. Hàm ***sanitize_remediation_code*** loại comment mẫu.
3. **Review** — AI tự review code so với ngữ cảnh; kết quả: ***approved*** hoặc ***corrected***.

**Cấu trúc resolution dict chính:**
- ***suggestion***, ***remediation_code***, ***classification***
- ***ai_status*** (***success*** | ***failed*** | ***fallback_mock***) và ***ai_error*** khi lỗi
- ***model***, ***generated_at***, ***remediation_target_file***

**Cache:** Kết quả cache theo SHA-256(***rule_id*** + ***code_text*** + ***surrounding_code***), TTL ***24h***. Cache hit bỏ qua pipeline.

**Fallback:** Thiếu API key / mock mode (***--mock-ai***) → ***fallback_mock***; timeout → ***failed***, không có ***remediation_code***.

### 3.5.4. Metrics: Độ phức tạp mã nguồn

Được tính cho mỗi file trong project, lưu tại ***report.metrics.files[rel_path]***, dùng để tạo findings, dashboard rating (A–E theo ISO 25010) và đánh giá threshold.

**Các chỉ số:**
- ***complexity*** — CCN (Lizard), đếm đường đi độc lập trong control flow
- ***cognitive_complexity*** — theo SonarQube, cao hơn CCN vì phạt nesting
- ***loc*** — dòng code thực (Lizard)
- ***level*** — ***LOW*** (≤10), ***MEDIUM*** (10–ngưỡng), ***HIGH*** (>ngưỡng)
- ***functions*** — chi tiết từng hàm: ***name***, ***start_line***, ***end_line***, ***complexity***, ***loc***

**Quy trình:** ***_compute_metrics*** duyệt source files → ***analyze_file_complexity*** (Lizard + cognitive) → sinh findings nếu cấp độ cao → phát hiện trùng lặp (***gen_duplication_findings***).

**Ngưỡng:** ***--ccn-threshold*** (mặc định 25) và ***--cognitive-threshold*** (mặc định 15).

### 3.5.5. Lưu trữ và tổ chức file

Toàn bộ dữ liệu được lưu dưới dạng file JSON trong thư mục ***~/.vexcode/***. Cấu trúc thư mục:
- ***~/.vexcode/reports/*** — Chứa báo cáo scan, mỗi project có một thư mục con:
  - ***report_YYYY-MM-DD-HH-mm-ss.json*** — Báo cáo chi tiết dạng VexCode
  - ***report_YYYY-MM-DD-HH-mm-ss.sarif*** — Báo cáo chuẩn SARIF (sidecar)
  - ***.latest*** — Pointer đến file báo cáo mới nhất
- ***~/.vexcode/backups/*** — Lưu backup file trước khi apply fix, tên theo đường dẫn đã encode
- ***~/.vexcode/ai_cache/*** — File ***ai_resolutions.json*** lưu cache AI resolution, TTL 24h
- ***~/.vexcode/apikey*** — API key Bearer token (32-byte hex), quyền ***0o600***
- ***~/.vexcode/.env*** — Cấu hình AI provider (API key, base URL, model)

Các file JSON được giữ lại vĩnh viễn để hỗ trợ so sánh giữa các lần quét (trend analytics): không có cơ chế xóa tự động. Mỗi file báo cáo là độc lập, không có quan hệ khóa ngoại, giúp đơn giản hóa việc sao lưu và di chuyển dữ liệu.

Cấu hình AI provider được lưu trong file ***.env*** tại ***packages/engine/***, đọc bởi Python engine khi scan và ghi bởi Express server khi người dùng thay đổi cấu hình qua Web UI.

### 3.5.6. Mô hình miền

Hệ thống sử dụng mô hình dữ liệu phi quan hệ, dựa trên file JSON thay vì database server. Các thực thể chính:
- **Finding**: Đơn vị dữ liệu cốt lõi, được sinh từ các scanner (OpenGrep, Gitleaks, OSV) và được bổ sung thông tin qua các giai đoạn Enrich và Resolve.
- **Report**: Bản tóm tắt một phiên scan, chứa danh sách findings và metadata về thời gian, mục tiêu, trạng thái Git.
- **Resolution**: Kết quả phân loại và đề xuất sửa lỗi từ AI, liên kết với Finding qua ***rule_id***.
- **Metrics**: Dữ liệu về độ phức tạp mã nguồn, được tính riêng cho từng file và tổng hợp vào Report.

Tất cả dữ liệu này được lưu trữ phân tán dưới dạng file JSON trong ***~/.vexcode/reports/***, không có khóa ngoại hay ràng buộc quan hệ. Việc này giúp hệ thống dễ bảo trì, dễ sao lưu và không phụ thuộc vào dịch vụ database bên ngoài.
