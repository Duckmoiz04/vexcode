## 4.5. Triển khai REST API

Hệ thống triển khai 14 REST API endpoints, tất cả đều trả về JSON. Các endpoints được tổ chức theo RESTful convention với resource-based URLs.

API config sử dụng GET/PUT trên cùng một URL để đọc và ghi cấu hình. API scan sử dụng SSE cho real-time progress, với endpoint `/api/scan/stream` trả về Content-Type `text/event-stream`. API reports sử dụng nested routes với project name và report ID trong URL path.

Mỗi API endpoint đều có error handling với try/catch, trả về HTTP status code phù hợp (200 cho thành công, 400 cho bad request, 500 cho server error) kèm JSON error message.
