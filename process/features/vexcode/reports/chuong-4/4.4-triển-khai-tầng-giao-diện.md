## 4.4. Triển khai tầng giao diện

### 4.4.1. State Management

Trạng thái toàn bộ ứng dụng được quản lý tập trung trong `context/AIProviderContext.tsx`. Context này cung cấp state cho findings, reports, projects, config, scan status, và filter state.

Context triển khai các action function để cập nhật state:
- `setCurrentReport(report)`: cập nhật findings list, current report, và trigger load lại projects/history.
- `updateConfig(config)`: cập nhật AI provider config và gọi API PUT để lưu xuống file .env.
- `setFilter(severities, categories, searchQuery)`: cập nhật filter state, các component consume context sẽ re-render tự động.
- `startScan()`: thiết lập scan status, mở kết nối SSE.
- `handleScanComplete(report)`: cập nhật findings, chuyển tab, đóng scan modal.

### 4.4.2. Custom Hooks

Logic nghiệp vụ được đóng gói trong các custom hook:

`useScan.ts` quản lý toàn bộ vòng đời của một phiên quét. Hook này tạo EventSource kết nối đến `/api/scan/stream`, lắng nghe các sự kiện progress, complete, error. Khi nhận sự kiện progress, hook cập nhật scanProgress và scanLogs trong context. Khi nhận sự kiện complete, hook parse dữ liệu report và gọi handleScanComplete. Khi nhận sự kiện error, hook cập nhật scanStatus thành 'error'. Hook cũng cung cấp hàm cancelScan để hủy scan đang chạy.

`useConfig.ts` quản lý cấu hình AI provider. Hook này đọc config từ API khi component mount, cung cấp hàm updateConfig để lưu thay đổi.

`useReports.ts` quản lý danh sách projects và reports. Hook này gọi API để lấy danh sách project, lấy danh sách report cho một project, và lấy nội dung chi tiết của một report.

`useChat.ts` quản lý chat với AI về finding cụ thể. Hook này lưu trữ lịch sử chat, gọi API `/api/chat` khi người dùng gửi câu hỏi.

`useAutoScroll.ts` tự động scroll component đến vị trí được chỉ định, sử dụng cho Code Inspector để highlight dòng lỗi.

`useToast.ts` quản lý thông báo tạm thời (toast notifications) cho các thao tác thành công hoặc thất bại.

### 4.4.3. Components

Components được tổ chức theo domain, mỗi component nằm trong thư mục riêng kèm test file:

**Header** chứa navigation tabs (Overview, Issues, Security Hotspots, Activity, Graph) và Scan Button. Navigation tabs sử dụng activeTab state từ context để highlight tab đang chọn. Scan Button trigger scan modal.

**Sidebar** hiển thị danh sách project và scan history. Sidebar lấy dữ liệu từ context và cho phép người dùng chọn report để xem. Sidebar cũng hiển thị filter counts cho từng category.

**Settings Drawer** là component phức tạp nhất, với các section con: AiProviderSelector cho phép chọn provider, ApiConfigSection nhập API key và base URL, ModelSelector chọn model, AdvancedSettings cho temperature và max tokens, QualityGateSection cấu hình ngưỡng chất lượng, SemgrepSection cấu hình rules.

**Dashboard** hiển thị Overview page với các MetricsCards (tổng số findings, số lỗi security, reliability, maintainability, performance), CategoryBreakdown (phân bố lỗi theo category), HealthScoreChart (điểm chất lượng theo thời gian), Leaderboards (top file có nhiều lỗi nhất), và CrossScanSummary (so sánh kết quả giữa các lần scan).

**Code Inspector** hiển thị mã nguồn với dòng lỗi được highlight sử dụng CodeMirror. Inspector có ThemePicker để chọn theme, FileViewer để xem code, ChatPanel để hỏi AI, DiffViewer để xem diff giữa code gốc và bản sửa, ApplyFixButton để áp dụng bản sửa.

**Filter Panel** cung cấp filter controls cho Issues page: lọc theo severity (error, warning, info), category (security, reliability, maintainability, performance), và search text.

**Scan Modal** hiển thị tiến độ quét với 6 bước, mỗi bước có trạng thái pending, active, completed được cập nhật real-time qua SSE.

### 4.4.4. Services Layer

`services/api.ts` cung cấp các hàm gọi REST API: `getConfig()`, `updateConfig()`, `startScan()`, `getReports()`, `getReport()`, `applyFix()`, `chatWithAI()`, `getFileContent()`.

`utils/reportNormalizer.ts` chuyển đổi dữ liệu report từ format nội bộ của engine sang format mà frontend sử dụng, bao gồm việc chuẩn hóa severity, category mapping, và sắp xếp findings.

`utils/sarifAdapter.ts` chuyển đổi findings từ SARIF format (Opengrep output) sang format nội bộ.

`utils/syntaxHighlight.tsx` triển khai syntax highlighting cho code blocks.
