## 4.2. Triển khai tầng CLI và REST API

### 4.2.1. CLI Entry Point

Điểm vào của CLI được triển khai trong file `bin/cli.js`. Module này xử lý ba command chính thông qua cơ chế phân tích tham số dòng lệnh thủ công, không sử dụng thư viện CLI bên ngoài.

Command `analyze` nhận tham số --target chỉ định đường dẫn dự án cần quét. Nếu không có --target, CLI hiển thị thông báo lỗi và hướng dẫn sử dụng. Tham số --fast kích hoạt chế độ quét nhanh, chỉ quét các file đã thay đổi trong git. Tham số --mock-scan và --mock-ai cho phép chạy ở chế độ offline, sử dụng dữ liệu giả lập. Khi chạy, CLI spawn Python process với các tham số tương ứng và in kết quả ra stdout dưới dạng JSON.

Command `serve` khởi động Express server. Tham số --port cho phép chỉ định cổng, mặc định là 3000. Server phục vụ REST API và static files từ thư mục `public/`, nơi chứa web dashboard đã được build sẵn.

Command `ui` kết hợp cả serve và tự động mở trình duyệt tại địa chỉ localhost. Trên Windows, lệnh sử dụng `start` để mở trình duyệt mặc định; trên Linux dùng `xdg-open`; trên macOS dùng `open`.

### 4.2.2. Express Server

Server được khởi tạo trong `src/server.js` thông qua hàm `createApp()`. Hàm này trả về một Express app instance đã được cấu hình sẵn với các middleware: CORS, JSON body parser, và static file serving. Các route được mount theo từng module riêng:

Route `/api/config` xử lý GET để đọc cấu hình AI provider từ file .env và PUT để ghi cấu hình mới. Cấu hình được parse từ file .env thông qua hàm `readEnvConfig()` trong fileService. Khi PUT, dữ liệu được ghi vào file .env thông qua `writeEnvConfig()`. Server trả về JSON với các trường provider, apiKey (masked), baseUrl, model.

Route `/api/scan/stream` triển khai Server-Sent Events cho real-time progress tracking. Khi nhận request GET, server thiết lập kết nối SSE với headers `Content-Type: text/event-stream` và `Cache-Control: no-cache`. Server spawn Python subprocess thông qua bridge.js, lắng nghe stdout của process để parse progress events, và gửi từng event đến client. Mỗi event có dạng `data: {"type": "progress", "step": 1, "message": "..."}`. Khi scan hoàn tất, server gửi event `type: "complete"` kèm toàn bộ dữ liệu report.

Route `/api/scan` (POST) thực hiện scan đồng bộ, spawn Python process và chờ kết quả. Route `/api/scan/cancel` (POST) kill Python process tree.

Route `/api/reports` trả về danh sách project dựa trên cấu trúc thư mục trong `~/.vexcode/reports/`. Route `/api/report/:project/:id` đọc file JSON từ thư mục tương ứng và trả về nội dung. Route `/api/re-resolve` cho phép chạy lại AI resolution trên report cũ mà không cần quét lại.

Route `/api/apply` nhận finding ID và nội dung bản sửa, tạo backup file tại `~/.vexcode/backups/`, sau đó ghi nội dung sửa vào file gốc.

Route `/api/chat` nhận câu hỏi của người dùng về một finding cụ thể và gọi AI API để trả lời.

### 4.2.3. Python Bridge

Module `src/bridge.js` đóng vai trò cầu nối giữa Node.js và Python. Bridge sử dụng `child_process.spawn` để chạy Python script với tham số dòng lệnh. Trên Windows, shell được bật (shell: true) để đảm bảo tương thích.

Khi scan, bridge spawn Python process với tham số `--target`, `--mock-scan`, `--mock-ai`, và `--output` chỉ định đường dẫn file JSON kết quả. Bridge lắng nghe sự kiện stdout và stderr của process để parse progress log. Mỗi dòng stdout có định dạng `[PROGRESS] step_id|message` được parse và chuyển thành SSE event.

Bridge hỗ trợ cancel scan bằng cách gọi `process.kill()` với signal SIGTERM trên Linux/macOS và `taskkill` trên Windows để đảm bảo toàn bộ process tree bị dừng.

### 4.2.4. Services Layer

Tầng services cung cấp các hàm tiện ích cho routes:

`fileService.js` cung cấp `isPathSafe()` kiểm tra đường dẫn nằm trong workspace boundary bằng case-insensitive prefix check, và các hàm đọc/ghi file .env. `backupService.js` quản lý backup files: tạo backup với timestamp trước khi apply fix, liệt kê danh sách backup, và restore file từ backup. `reportService.js` cung cấp các hàm `listProjects()`, `listProjectReports()`, `getReportContent()` để đọc cấu trúc thư mục reports.
