## 5.2. Kết quả kiểm thử

### 5.2.1. Kiểm thử pipeline phân tích (Engine)

Tầng engine gồm 19 test files viết bằng pytest, kiểm tra từng phase của pipeline: scanner, enricher, dedup, resolver, reporter.

ISO 25010 taxonomy tests kiểm tra bộ phân loại deterministic: CWE ID mapping (CWE-79 → Security/XSS), rule_id keyword mapping (`detect-child-process` → Security/RCE), OWASP ID normalization, context override cho style/naming convention, và edge cases (CWE không tồn tại, rule_id rỗng, message null).

Scanner tests kiểm tra fast scan detection (git status parsing, repo sạch, file thay đổi) và OpenGrep scan execution. AI Resolver tests kiểm tra prompt building, API call với mock responses, retry logic, parallel execution, và mock fallback. Pipeline tests kiểm tra tích hợp giữa các phase: scanner + enricher, enricher + resolver, resolver + reporter. AST Graph tests kiểm tra GitNexus CLI giao tiếp: availability check, symbol resolution, context fetching, impact computation, và mock fallback khi GitNexus không available.

Đặc biệt, test normalize classification tại `ai_resolver.py` đảm bảo mọi giá trị ngoài spec (như `hotspot`) tự động ép về `confirmed` — 46/46 tests pass sau khi fix.

### 5.2.2. Kiểm thử tầng CLI

Tầng CLI có 5 test files viết bằng Vitest (107 tests). Server tests (63 tests) kiểm tra tất cả REST API endpoints: GET/PUT config, GET scan stream, POST scan, POST cancel, GET reports, GET report detail, POST re-resolve, POST apply fix, POST chat, GET file-content, GET/POST backup. Bridge tests (15 tests) kiểm tra Python subprocess spawning, stdout/stderr parsing, và graceful fallback khi Python không khả dụng. CLI tests (5 tests) kiểm tra argument parsing. File service tests (4 tests) kiểm tra path safety với path traversal, đường dẫn tuyệt đối, đường dẫn tương đối. Finding status tests kiểm tra tracking trạng thái (open, in_progress, resolved, dismissed).

### 5.2.3. Kiểm thử tầng giao diện

Tầng giao diện có 32 test files viết bằng Vitest với jsdom (224 tests). Context tests kiểm tra AIProviderContext: khởi tạo state, cập nhật findings/config, lọc findings, xử lý scan complete. Hook tests kiểm tra useScan (SSE handling, cancel, progress), useConfig, useReports, useChat, useToast, useAutoScroll. Component tests kiểm tra rendering và interaction cho Header, Sidebar, SettingsDrawer, Dashboard, Code Inspector, FilterPanel, ScanModal, Markdown renderer, ErrorBoundary. Utility tests kiểm tra reportNormalizer, sarifAdapter, syntaxHighlight.

### 5.2.4. Kiểm thử thực nghiệm trên ba dự án

Ngoài unit tests, hệ thống được kiểm thử thực nghiệm end-to-end trên ba dự án mã nguồn mở (Bảng 4.1). Kết quả tổng hợp:

| Chỉ số | VexCode | Juice Shop | OpenHands | Tổng |
|--------|---------|-----------|-----------|------|
| Tổng findings | 142 | 130 | 284 | 556 |
| Security findings | 25 | 73 | 107 | 205 |
| AI confirmed | 113 | 127 | 215 | 455 |
| AI false_positive | 29 | 3 | 69 | 101 |
| AST context enriched | 43 (30%) | 0 | 0 | 43 |

Kết quả thực nghiệm cho thấy pipeline hoạt động ổn định, phát hiện đúng loại lỗ hổng trên cả ba dataset đa quy mô và đa ngôn ngữ. AI Resolution phân loại chính xác false_positive cho linting rules (`no-print-statements`, `no-silent-except`), phản ánh khả năng hiểu ngữ cảnh của LLM. Deep Context Analysis cung cấp AST enrichment cho 30% findings tại dataset có GitNexus installed.
