## 5.3. Đánh giá theo ISO/IEC 25010

Hệ thống được đánh giá dựa trên các tiêu chí ISO/IEC 25010 đã xác định trong Chương 2, sử dụng bằng chứng từ thử nghiệm thực nghiệm (Chương 4).

### 5.3.1. Functional Suitability — Tốt

Hệ thống đáp ứng đầy đủ yêu cầu chức năng của cả hai mục tiêu đồ án.

**Mục tiêu 1** — Deep Context-Aware Analysis: GitNexus CLI lập chỉ mục AST và truy xuất ngữ cảnh symbol (callers, blast_radius) cho 43/142 findings (30%) tại dataset VexCode. Khi GitNexus unavailable, pipeline không crash — kết quả thử nghiệm trên juice-shop và OpenHands (0 AST context) xác nhận fallback graceful.

**Mục tiêu 2** — Đánh giá đa tiêu chuẩn và đề xuất tối ưu: Pipeline đa-scanner (OpenGrep + Gitleaks + OSV) phát hiện 556 findings trên ba dự án, phân loại theo ISO 25010 (Security 205, Maintainability 304, Operability 47). AI Resolution 3-stage phân loại 2 nhãn (confirmed/false_positive) với Smart Gate nhóm theo rule_id — chỉ gọi AI cho 5 unique rules/lần chạy. AI fix suggestion đạt 100% success (2/2 fix, 2/2 review approved) trên dataset VexCode.

Dashboard hiển thị findings list, dependency graph (Sigma.js), dataflow trace, và chat AI — tất cả hoạt động với real-time SSE progress tracking. OSV scanner phát hiện thêm 28 dependency vulnerabilities trên juice-shop mà OpenGrep không bao phủ.

### 5.3.2. Reliability — Tốt

Hệ thống duy trì ổn định trong mọi kịch bản thử nghiệm:

- **Fallback khi thiếu API key**: AI resolver chuyển sang mock mode, pipeline tiếp tục, không crash.
- **Fallback khi GitNexus unavailable**: findings không có `ast_context`, pipeline hoàn tất bình thường.
- **LLM timeout (>30s)**: AI resolver trả về `failed`, finding giữ status `confirmed`, không block pipeline.
- **Atomic write report**: Reporter sử dụng ghi nguyên tử để tránh file corrupted khi crash.
- **0 crash trong 10 lần scan liên tiếp** trên ba dataset.

### 5.3.3. Security — Tốt

Kết quả thử nghiệm bảo mật (Bảng 4.5) cho thấy hệ thống chặn đầy đủ 5 vector tấn công: path traversal, payload quá lớn, brute force, thiếu Bearer token, và thiếu API key. Bên cạnh bảo mật API, AI Resolution còn phân loại chính xác false_positive cho linting rules — tránh làm lãng phí thời gian developer.

Hệ thống phát hiện đúng loại lỗ hổng bảo mật trên cả ba dataset: XSS (Jinja2 template, `res.write()`), SQL injection (Sequelize), RCE (`child_process`), JWT hardcoded/unverified, path traversal, credential leak, insecure WebSocket, và Docker security.

### 5.3.4. Performance Efficiency — Khá

AI resolution chạy song song với ThreadPoolExecutor (`max_workers=3` mặc định). Enrichment sử dụng 10 parallel workers. AI cache SHA-256 đạt 50% hit rate trên dataset VexCode (3/6 cache hits), giảm số lượng API call повтор.

Smart Gate nhóm findings theo rule_id và chỉ gọi AI cho unique rules — trên mỗi dataset chỉ 5 rules được resolve, thay vì N calls cho N findings. Điều này giảm đáng kể số lượng API call và chi phí.

Giới hạn: report thiếu trường `scan_duration_seconds`, chưa đo thời gian mỗi phase pipeline, chưa benchmark trên dự án lớn (>10.000 file).

### 5.3.5. Usability — Khá

Giao diện CLI cung cấp ba command chính với `--help`. Web Dashboard có layout rõ ràng: header navigation, sidebar findings list, main content area với Code Inspector. Filter panel lọc theo severity, category, search text. Dependency graph hiển thị trực quan mối liên kết file → symbol → callers → blast_radius bằng Sigma.js + ForceAtlas2 layout. Dataflow trace hiển thị source → propagators → sink cho security findings. Chat AI cho phép hỏi đáp về findings. Apply Fix áp dụng bản sửa một click, kèm backup tự động.

Hạn chế: chưa có hướng dẫn sử dụng cho người mới, chưa hỗ trợ export PDF, chat chưa có multi-turn context sâu.

### 5.3.6. Maintainability — Khá

Kiến trúc module hóa 5 phase (Scanner → Enricher → Dedup → Resolve → Report) cho phép thay thế từng thành phần độc lập. Node.js CLI giao tiếp Python engine qua subprocess bridge, không coupling trực tiếp. React Dashboard quản lý state tập trung qua AIProviderContext, components chỉ consume context qua hooks.

Hạn chế: module `ai_resolver.py` ~1003 dòng — vượt ngưỡng 200 dòng của tiêu chuẩn dự án. Chưa có API documentation tự động (Swagger/OpenAPI). Chưa có CI/CD pipeline.
