# UX/UI Context

This file is the canonical UX/UI context entrypoint for AI Code Review.

Use it after `process/context/all-context.md` when the task needs UI styling guidelines, design system rules, or component architecture.

---

## Scope

This group covers:
- Core UI styling rules and aesthetics (vanilla CSS, harmonious palettes).
- Design system tokens (colors, typography, spacing).
- local Web UI layout and design preferences.

It does not cover:
- CLI console output formatting (belongs in general coding conventions).
- Backend APIs or routing (belongs in general context).

## Read When

Read this entrypoint when:
- Designing or modifying UI components for the local Web UI.
- Adjusting CSS styles, color themes, or visual effects.
- Implementing configuration screens in the Web UI.

## Quick Routing

- use `process/context/uxui/uiux.md` for core design guidelines, aesthetic standards, and styling tokens.

## Source Paths

- `process/context/uxui/uiux.md`

## Update Triggers

Update this group when:
- Visual guidelines, theme palettes, or typography systems change.
- A new UI component library or CSS framework is introduced.
